# Contributing to DOM Wallet V3

## Language policy

All source code, comments, documentation, specifications, commit messages, pull requests, issues, generated reports, test names, and technical artifacts must be written in English.

## Authorship policy

All commits must be authored and committed as:

`Soren Planck <sorenplanck@tutamail.com>`

Co-authored-by trailers are prohibited. AI tools may assist with engineering work but must not be listed as authors, co-authors, committers, maintainers, or contributors.

## Engineering-reference policy

DOM Wallet V3 preserves the philosophy, consensus, cryptography, privacy, chain identity, wire formats, and economic rules of the DOM Protocol.

Existing DOM Wallet V1 and V2 components must be evaluated and retained when they are correct, compatible, and demonstrably valuable.

Epic Wallet may be studied for architectural strategies, protected properties, failure handling, recovery techniques, and testing methods. Epic source code, tests, comments, identifiers, formats, constants, and implementation-specific expressions must not be copied.

The governing rule is:

`DOM semantics > Epic strategy`

## Development policy

Every implementation must be independently written for DOM and must be linked to an approved specification, explicit invariants, failure-path behavior, and reproducible tests.

No feature is complete based only on successful happy-path execution.
